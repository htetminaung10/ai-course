🎯 Intro to Supervised Classification
Classification assigns inputs to discrete categories. Unlike regression (predicting a number), classification predicts a class label — spam/not spam, cancer/benign, cat/dog.
⏱ ~35 min
🎯 2 Quizzes
⭐ 160 XP
🧩
Discriminative vs Generative
Two fundamentally different approaches to classification
🎯 Discriminative Models
Directly learn the decision boundary between classes by modelling P(y|x) — the probability of the class given the input.

Examples: KNN, Logistic Regression, SVM, Decision Trees

Ask: "Given these features, which class is it?"
🧬 Generative Models
Model the joint distribution P(x, y), then use Bayes' rule to get P(y|x).

Examples: Naive Bayes, Gaussian Mixture Models

Ask: "What does each class look like? Which class is this most likely from?"
🌍 Real World Example
Email Spam Detection at Gmail
Gmail's spam filter is a discriminative classifier. It learns a decision boundary between "spam" and "not spam" from millions of labelled emails. Given a new email's features (word frequencies, sender reputation, formatting), it outputs P(spam|email). If P > 0.5, it's spam. The model doesn't try to model what a "typical spam email looks like" — it just learns where the boundary is.

📏 K-Nearest Neighbours
The simplest classification algorithm imaginable: "You are who your neighbours are." KNN finds the k most similar training points and votes on the class. No training required.
⏱ ~30 min
🎯 2 Quizzes
⭐ 160 XP
🎮 Interactive
📐
How KNN Works
📐 The Algorithm — 3 Steps
1. Choose k — the number of neighbours to consider
2. Measure distance — find the k training points closest to the new point (usually Euclidean distance)
3. Vote — the most common class among those k neighbours wins

For regression: take the mean of k neighbours instead of majority vote.
🎮 Interactive KNN — click to classify a new point (green star)
k = 3
⬆️ Large k
Smoother decision boundary. More robust to noise. Higher bias, lower variance. Risk of underfitting. Slower prediction.
⬇️ Small k (k=1)
Jagged decision boundary. Very sensitive to noise. Low bias, high variance. Risk of overfitting. Fast prediction.
🌍 Real World Example
Netflix Content Recommendation
Netflix uses KNN-style logic for content recommendation: find the k users most similar to you (same viewing history, ratings, genre preferences), then recommend what they watched. "You watched Narcos — 5 of your 7 nearest neighbours also loved Sicario." Simple, interpretable, and surprisingly effective as a baseline.
Python — KNN with Sklearn
from sklearn.neighbors import KNeighborsClassifier
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler

# KNN is sensitive to scale — ALWAYS standardise first!
scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s  = scaler.transform(X_test)

# Train KNN (k=5 is a common starting point)
knn = KNeighborsClassifier(n_neighbors=5, metric='euclidean')
knn.fit(X_train_s, y_train)

# Predict
y_pred = knn.predict(X_test_s)
print(knn.score(X_test_s, y_test))  # accuracy

# Find optimal k using cross-validation
from sklearn.model_selection import cross_val_score
k_range = range(1, 21, 2)
cv_scores = [cross_val_score(KNeighborsClassifier(k=k), X, y, cv=5).mean()
             for k in k_range]
best_k = k_range[cv_scores.index(max(cv_scores))]

🌳 Decision Trees
A decision tree learns a hierarchical set of if-else rules directly from data. Highly interpretable, fast, and the building block for powerful ensemble methods like Random Forests.
⏱ ~40 min
🎯 2 Quizzes
⭐ 180 XP
🌿
Anatomy of a Decision Tree
🌿 Structure
Root node — top-level split using the most informative feature
Internal nodes — further splits (binary: left/right)
Branches — outcomes of each rule (True/False)
Leaf nodes — final class predictions

Each node tests one feature at a time against a threshold.
Example: Classifying loan applications (click nodes to explore)
Click any node to learn more
📐
How Trees Learn — Splitting Criteria
🎲 Gini Impurity
Measures how often a randomly chosen element would be incorrectly classified.

Gini = 1 − Σ pᵢ²

Gini = 0 → pure node (all one class) ✅
Gini = 0.5 → perfectly impure (50/50 split) ❌

Default in sklearn. Fast to compute.
🌡️ Information Gain (Entropy)
Measures the reduction in uncertainty after a split using Shannon entropy.

H = −Σ pᵢ log₂(pᵢ)

H = 0 → pure node (certain) ✅
H = 1 → maximally uncertain ❌

Slightly slower, often similar results.
🌍 Real World Example
Decision Trees in Medical Diagnosis
The CHADS₂ stroke risk score used by cardiologists is a hand-built decision tree: Congestive heart failure? +1. Hypertension? +1. Age 75+? +1. Diabetes? +1. Prior stroke? +2. Total score → risk level → treatment decision. ML-learned decision trees automate this process from patient data, discovering which features split risk groups most cleanly — and they can be printed as a flowchart for doctors to verify.
⚠️
Decision Trees overfit easily. An unconstrained tree will memorise every training point (every leaf has 1 sample). Always limit: max_depth, min_samples_leaf, or min_samples_split. Or use ensemble methods (Random Forest) to tame the variance.
Python — Decision Tree

    criterion='gini',       # or 'entropy'
    max_depth=4,            # prevent overfitting

tree.fit(X_train, y_train)

# Visualise the tree

📉 Logistic Regression
Despite the name, Logistic Regression is a classification algorithm. It wraps a linear model in a sigmoid function to output probabilities between 0 and 1.
⏱ ~40 min
🎯 2 Quizzes
⭐ 180 XP
🎮 Interactive
σ
The Sigmoid Function
Squashing any real number into [0,1]
Logistic Regression Model
p = σ(wx + b) = 1 / (1 + e^−(wx+b))
p = probability of class 1 · w = weight · b = bias · σ = sigmoid function
⚡ Interactive Sigmoid — adjust w and b
Weight (w): 1.0
Bias (b): 0.0
📊 Effect of Weight (w)
Large |w| → steep sigmoid (more confident boundary)
Small |w| → shallow sigmoid (more uncertain)
Negative w → reverses the direction

w controls the steepness of the decision boundary.
📍 Effect of Bias (b)
Shifts the sigmoid left or right.
Positive b → boundary shifts left (more likely to predict class 1)
Negative b → boundary shifts right

b controls the position of the decision boundary.
🌍 Real World Example
Fraud Detection at Barclays
Barclays uses logistic regression as a fast, interpretable fraud scoring layer. Input features: transaction amount, merchant category, time of day, distance from last transaction. Output: P(fraud) between 0 and 1. If P(fraud) > 0.85, trigger a real-time block. The model's weights are auditable ("large transaction weight = 0.3, foreign country weight = 0.4"), making it explainable to regulators — a key requirement under FCA rules.
Python — Logistic Regression
from sklearn.linear_model import LogisticRegression
from sklearn.preprocessing import StandardScaler

# Scale first — logistic regression uses gradient descent
scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s  = scaler.transform(X_test)

model = LogisticRegression(
    C=1.0,           # regularisation (1/lambda) — lower = more regularised
    max_iter=1000,
    random_state=42
)
model.fit(X_train_s, y_train)

# Get class labels AND probabilities
y_pred       = model.predict(X_test_s)        # [0, 1, 1, 0, ...]
y_prob       = model.predict_proba(X_test_s)   # [[0.8, 0.2], [0.1, 0.9], ...]
fraud_prob   = y_prob[:, 1]                   # probability of fraud

# Custom threshold (default 0.5)
y_custom = (fraud_prob > 0.85).astype(int)    # stricter threshold

# Multiclass (One-vs-Rest)
model_mc = LogisticRegression(multi_class='ovr')  # or 'multinomial'

⚖️ Model Validation
A model that memorises training data is useless. Validation techniques help you estimate real-world performance — and the bias-variance tradeoff explains why every model must balance two fundamental sources of error.
⏱ ~40 min
🎯 3 Quizzes
⭐ 200 XP
⚖️
The Bias-Variance Tradeoff
📌 High Bias (Underfitting)
Model too simple. Makes strong assumptions. Misses real patterns in the data.

Symptom: poor training AND test accuracy
Example: Linear model on non-linear data
Fix: More complex model, more features
🎲 High Variance (Overfitting)
Model too complex. Memorises training noise. Fails on new data.

Symptom: great training accuracy, poor test accuracy
Example: Deep tree with no depth limit
Fix: Regularisation, more data, simpler model
Bias-Variance as model complexity increases
🔀
Cross-Validation
Getting a reliable performance estimate with limited data
🔀 k-Fold Cross-Validation
Split data into k equal folds. Train on k-1 folds, test on the remaining fold. Repeat k times (each fold is the test set once). Average the k scores.

Typical: k=5 or k=10. Uses all data for both training and evaluation. More reliable than a single train/test split.
Python — Validation
from sklearn.model_selection import cross_val_score, StratifiedKFold
from sklearn.tree import DecisionTreeClassifier
import numpy as np

model = DecisionTreeClassifier(max_depth=5, random_state=42)

# Stratified k-fold: preserves class balance in each fold
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=cv, scoring='accuracy')

print(f"CV Accuracy: {scores.mean():.3f} ± {scores.std():.3f}")

# Learning curve — diagnose bias vs variance
from sklearn.model_selection import learning_curve
train_sizes, train_scores, val_scores = learning_curve(
    model, X, y,
    train_sizes=np.linspace(0.1, 1.0, 10),
    cv=5, scoring='accuracy'
)

🌍 Real World Example
The Accuracy Paradox — COVID Test
Imagine COVID affects 1% of a population of 10,000. A classifier that predicts "no COVID" for everyone achieves 99% accuracy — but has 0% recall. It misses every single case. A naive accuracy metric would call this an excellent model. Precision and recall expose the truth: the model is completely useless for its intended purpose. This is why accuracy alone is never sufficient for imbalanced medical/fraud datasets.

📈 ROC Curves & AUC
The ROC curve shows how precision and recall trade off at every possible classification threshold. AUC summarises this into a single number — a threshold-independent measure of model quality.
⏱ ~35 min
🎯 2 Quizzes
⭐ 180 XP
🎮 Interactive
📈
The ROC Curve
Receiver Operating Characteristic
📈 What the ROC Curve Shows
By varying the classification threshold from 0 to 1, every threshold gives a different (TPR, FPR) pair:

TPR (True Positive Rate = Recall) = TP / (TP + FN) — how many real positives we catch
FPR (False Positive Rate) = FP / (FP + TN) — how many negatives we falsely flag

Plot all these pairs → the ROC curve. The Area Under the Curve (AUC) summarises performance across all thresholds.

🌍 Real World Example
Credit Scoring at Experian
Experian's credit default prediction model is evaluated by AUC across millions of applications. AUC is threshold-independent, which matters because different lenders choose different thresholds based on their risk appetite — a conservative bank might set threshold=0.7, a fintech lender might use 0.4. AUC measures the model's intrinsic quality regardless of where you draw the line. An AUC of 0.88 means: randomly pick a defaulter and a non-defaulter — the model ranks the defaulter higher 88% of the time.
