🔧 Sklearn Pipelines
A Pipeline chains preprocessing and modelling steps into a single object. This eliminates data leakage, makes code reproducible, and enables one-line cross-validation on the full workflow.
⏱ ~35 min
🎯 2 Quizzes
⭐ 170 XP
🎮 Interactive
🔗
Why Pipelines?
The three problems they solve
🛡️ Prevent Leakage
Without a pipeline, you might fit your scaler on the full dataset (including test), causing data leakage. A Pipeline ensures fit_transform only on train, transform only on test — automatically.
🔁 One-Line CV
With a Pipeline, cross_val_score(pipeline, X, y) correctly applies preprocessing separately inside each fold. Without it, you'd have to manually split and preprocess — error-prone.
📦 Deployment
A fitted Pipeline is a single pickle file. Deploy it and call pipeline.predict(raw_data) — it handles all preprocessing automatically on new data.

🌍 Real World Example
Deployment at a UK Fintech
A loan approval model at Funding Circle uses a sklearn Pipeline with: (1) ColumnTransformer — StandardScaler for numeric features, OneHotEncoder for categoricals, (2) FeatureUnion joining engineered features, (3) RandomForestClassifier. The entire pipeline is serialised to a single file at training time. When a loan application arrives, one call to pipeline.predict_proba(application) returns the approval probability — all preprocessing is embedded, no separate preprocessing code in production.
Python — Full Pipeline with ColumnTransformer
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.impute import SimpleImputer
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import cross_val_score
import pickle

# Define feature groups
numeric_features  = ['age', 'income', 'loan_amount']
categorical_features = ['employment_type', 'region']

# Preprocessing for each feature type
numeric_transformer = Pipeline([
    ('imputer', SimpleImputer(strategy='median')),
    ('scaler',  StandardScaler()),
])
categorical_transformer = Pipeline([
    ('imputer', SimpleImputer(strategy='most_frequent')),
    ('encoder', OneHotEncoder(handle_unknown='ignore')),
])

# Combine with ColumnTransformer
preprocessor = ColumnTransformer([
    ('num', numeric_transformer,  numeric_features),
    ('cat', categorical_transformer, categorical_features),
])

# Full pipeline: preprocessing + model
pipeline = Pipeline([
    ('preprocessor', preprocessor),
    ('classifier',  RandomForestClassifier(n_estimators=200, random_state=42)),
])

# Cross-validate the FULL pipeline (no leakage!)
scores = cross_val_score(pipeline, X, y, cv=5, scoring='roc_auc')
print(f"CV AUC: {scores.mean():.3f} ± {scores.std():.3f}")

# Train final pipeline and save
pipeline.fit(X_train, y_train)
with open('model.pkl', 'wb') as f:
    pickle.dump(pipeline, f)  # entire preprocessing embedded!

🏠 Validating on Hierarchical Data
Real data is rarely independent. When rows are grouped (patients in hospitals, trips in households, transactions per customer), random splitting leaks information between train and test — inflating your performance estimates.
⏱ ~30 min
🎯 2 Quizzes
⭐ 160 XP
🎮 Interactive
🏘️
The Problem with Random Splits
The travel survey example
🏘️ Nested Data Structure
A travel survey records multiple trips per household. Members of the same household:
• Live in the same area (same commute distance)
• Share the same car/bike ownership
• Have similar travel preferences

A random train/test split will place different trips from the same household in both sets. The model learns household-specific patterns (memorisation), then "predicts" them on the test set — artificially inflating accuracy.
🎮 Compare Split Strategies — click to toggle
❌ Random Split (WRONG)
Trips from same household appear in both train and test:
⚠️ Estimated accuracy: 94% (artificially high)
✅ Group (Household) Split
Entire households go to either train or test — never split:
✅ Honest accuracy: 81% (realistic)
🌍 Real World Example
Clinical Trial Analysis — Patient-Level Split
A hospital built a model to predict patient readmission from lab tests. Using random row-level splits, the model achieved 92% AUC. The team then realised patients had multiple visits — the same patient was in both train and test. Switching to patient-level splits (GroupKFold on patient ID), the true AUC was 74%. The model had learned "which patient tends to be readmitted" rather than "what patterns predict readmission." Clinical deployment based on the 92% estimate would have been a costly failure.
Python — Group K-Fold
from sklearn.model_selection import GroupKFold, cross_val_score
from sklearn.ensemble import RandomForestClassifier
import pandas as pd

# Groups: one ID per household (never split across folds)
groups = df['household_id']   # or patient_id, customer_id, etc.

cv = GroupKFold(n_splits=5)
model = RandomForestClassifier(random_state=42)

scores = cross_val_score(
    model, X, y,
    groups=groups,  # KEY: ensures same group never in train AND test
    cv=cv,
    scoring='accuracy'
)
print(f"Honest CV: {scores.mean():.3f} ± {scores.std():.3f}")

# For train/test split with groups:
from sklearn.model_selection import GroupShuffleSplit
gss = GroupShuffleSplit(n_splits=1, test_size=0.2, random_state=42)
train_idx, test_idx = next(gss.split(X, y, groups=groups))

🔍 Hyperparameter Tuning
Every algorithm has dials you can turn. Hyperparameter tuning is the systematic search for the best dial settings — using Grid Search, Random Search, or smarter methods like Bayesian Optimisation.
⏱ ~35 min
🎯 2 Quizzes
⭐ 180 XP
🎮 Interactive
🗺️
Search Strategies
📐 Grid Search
Exhaustively try every combination in a predefined grid. Guaranteed to find the best in the grid — but scales exponentially. 5 values × 5 params = 5⁵ = 3,125 fits.
🎲 Random Search
Sample random combinations from the space. Often finds near-optimal in fewer trials. Works better in high dimensions because it doesn't waste time on less-important hyperparameters.
🧠 Bayesian Optimisation
Builds a probabilistic model of the objective function. Focuses on promising regions. Most efficient — typically finds near-optimal in 20-50 trials vs thousands for Grid Search.

Python — Grid, Random & Bayesian Search
from sklearn.model_selection import GridSearchCV, RandomizedSearchCV
from sklearn.ensemble import RandomForestClassifier
from scipy.stats import randint, uniform

rf = RandomForestClassifier(random_state=42)

## 1. Grid Search — exhaustive but expensive
param_grid = {
    'n_estimators':    [50, 100, 200],
    'max_depth':       [5, 10, 15, None],
    'min_samples_leaf':[1, 5, 10],
}  # 3×4×3 = 36 fits × 5 folds = 180 model fits!
grid = GridSearchCV(rf, param_grid, cv=5, scoring='roc_auc', n_jobs=-1)
grid.fit(X_train, y_train)
print(grid.best_params_)

## 2. Random Search — usually more efficient
param_dist = {
    'n_estimators':    randint(50, 500),
    'max_depth':       randint(3, 20),
    'min_samples_leaf':randint(1, 20),
    'max_features':    uniform(0.1, 0.9),
}
rand = RandomizedSearchCV(rf, param_dist, n_iter=50, cv=5,
                           scoring='roc_auc', random_state=42)

## 3. Bayesian Optimisation (optuna)
import optuna

def objective(trial):
    params = {
        'n_estimators': trial.suggest_int('n_estimators', 50, 500),
        'max_depth':    trial.suggest_int('max_depth', 3, 20),
    }
    model = RandomForestClassifier(**params)
    return cross_val_score(model, X, y, cv=3).mean()

study = optuna.create_study(direction='maximize')
study.optimize(objective, n_trials=50)

✂️ Feature Selection & Optimisation
More features isn't always better. Irrelevant features add noise, slow training, and hurt generalisation. Feature selection identifies which features to keep — using statistics, model-based ranking, or iterative search.
⏱ ~35 min
🎯 2 Quizzes
⭐ 170 XP
🎮 Interactive
📊
Three Categories of Feature Selection
🔬 Filter Methods
Select features before fitting any model, using statistical tests.

• Correlation with target
• Chi-squared test
• Mutual information

Fast but ignores interactions between features.
🔁 Wrapper Methods
Try different feature subsets by training and evaluating a model on each.

• Recursive Feature Elimination (RFE)
• Forward/backward selection

Slower but model-aware. Can overfit to training data.
🌲 Embedded Methods
Feature selection happens during model training.

• Lasso (L1 regularisation)
• Random Forest importances
• XGBoost feature scores

Best of both: fast and model-aware.

🌍 Real World Example
Credit Scoring — From 200 to 15 Features
Experian's credit models start with 200+ raw features. After feature selection: (1) Filter — remove features with near-zero correlation to default; (2) Lasso regression — L1 penalty automatically zeros out irrelevant features; (3) RFE — iteratively remove the weakest remaining feature until performance drops. The final model uses 15-20 features. Benefits: 10× faster inference, easier regulatory audit, less data collected per applicant, and — crucially — higher generalisation accuracy because noise features are removed.
Python — Feature Selection
from sklearn.feature_selection import (
    SelectKBest, f_classif, mutual_info_classif,  # filter
    RFE, RFECV,                                   # wrapper
    SelectFromModel                                # embedded
)
from sklearn.linear_model import LassoCV
from sklearn.ensemble import RandomForestClassifier
import pandas as pd

## 1. Filter — top K features by statistical test
selector = SelectKBest(score_func=mutual_info_classif, k=15)
X_filtered = selector.fit_transform(X_train, y_train)
selected_features = feature_names[selector.get_support()]

## 2. Wrapper — Recursive Feature Elimination with CV
rfecv = RFECV(
    estimator=RandomForestClassifier(n_estimators=50),
    step=1,           # remove 1 feature at a time
    cv=5,
    scoring='roc_auc'
)
rfecv.fit(X_train, y_train)
print(f"Optimal features: {rfecv.n_features_}")

## 3. Embedded — Lasso (L1) automatically zeros unimportant features
lasso = LassoCV(cv=5).fit(X_train, y_train)
important = pd.Series(lasso.coef_, index=feature_names)
selected = important[important != 0].index.tolist()

## In a Pipeline (recommended!)
pipeline = Pipeline([
    ('selector',    SelectFromModel(RandomForestClassifier(n_estimators=50))),
    ('classifier', RandomForestClassifier(n_estimators=200)),
])
