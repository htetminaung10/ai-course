🌐 Intro to Unsupervised Learning
What happens when you have data but no labels? Unsupervised learning discovers hidden structure — clusters, patterns, and representations — entirely from the data itself.
⏱ ~25 min
🎯 2 Quizzes
⭐ 120 XP
🆚
Supervised vs Unsupervised
The fundamental difference
✅ Supervised Learning
You provide labelled examples — input X paired with the correct output y. The model learns to predict y from X.

Examples: spam detection (spam/not spam), house price prediction (£ value), image classification (cat/dog).
🔍 Unsupervised Learning
You provide data only — no labels. The model finds hidden patterns, groupings, or a compressed representation.

Examples: customer segmentation, anomaly detection, topic modelling, dimensionality reduction.
💡
The vast majority of real-world data is unlabelled. Labelling is expensive and time-consuming. Unsupervised methods let you extract value from raw data without paying for annotation.
🗂️
The Three Families
What unsupervised learning can do
⚙️
Clustering
Group similar data points together. Each point belongs to a cluster.

Algorithms: K-Means, Hierarchical, DBSCAN, GMM
Use: Customer segments, document groups, image regions
📉
Dimensionality Reduction
Compress high-dimensional data into fewer dimensions while keeping structure.

Algorithms: PCA, t-SNE, UMAP, Autoencoders
Use: Visualisation, feature extraction, compression
🚨
Anomaly Detection
Find data points that don't fit the normal pattern — outliers.

Algorithms: Isolation Forest, DBSCAN noise points, Autoencoders
Use: Fraud, network intrusion, manufacturing defects

The Challenges
Why unsupervised learning is hard
📏 No Ground Truth
Without labels, how do you know if your clusters are meaningful? You can't just compute accuracy. You need domain knowledge and internal metrics like silhouette score or inertia.
🔢 Choosing K
Most clustering algorithms need you to specify the number of clusters upfront. But in real problems you often don't know — and the elbow method and silhouette analysis are heuristics, not guarantees.
📐 Scale Sensitivity
Many algorithms (K-Means, hierarchical) use distance metrics. A feature measured in £millions will dominate one measured in years. Always scale your features before clustering (StandardScaler).
🎲 Local Minima
K-Means converges to local optima — run it multiple times with different random seeds (n_init=10) and take the best result. The result depends on initialisation.
🌍 Real World Example
Netflix Content Strategy — Clustering Without Labels
Netflix uses unsupervised clustering on viewing behaviour to discover "taste communities" — groups of users who watch similar genres in similar patterns. These clusters didn't exist before the analysis; Netflix discovered them from raw watch data. The clusters power recommendations, marketing campaigns, and inform which original content to commission. They don't label users as "cluster A" — the clusters are internal representations that help the business, not ground truth categories.

⚙️ K-Means Clustering
K-Means is the most widely used clustering algorithm. It partitions data into K groups by iteratively assigning points to the nearest centroid and recomputing centroids — simple, fast, and powerful.
⏱ ~35 min
🎯 2 Quizzes
⭐ 160 XP
🎮 Interactive
🔄
The Algorithm — Step by Step
How K-Means converges to a solution
🎮 Interactive K-Means — Watch the algorithm converge
Click Step to advance one iteration, or Run All to animate.
Step 0 — Initial data. Centroids placed randomly.
Cluster 1 Cluster 2 Cluster 3 Centroid ✕
1️⃣
Initialise
Randomly place K centroids in the feature space. K is chosen by you before training.
2️⃣
Assign
Assign each data point to the nearest centroid using Euclidean distance. This forms K clusters.
3️⃣
Update → Repeat
Move each centroid to the mean of its cluster. Repeat assign + update until centroids stop moving.
💡
Convergence is guaranteed — inertia (sum of squared distances to centroids) strictly decreases each iteration. But it may converge to a local minimum, not the global optimum. This is why n_init=10 runs the algorithm 10 times and picks the best.
📐
Choosing K — The Elbow Method
How many clusters should you use?
The biggest challenge in K-Means is choosing K before you start. The elbow method plots inertia (within-cluster sum of squares) against K. As K increases, inertia always decreases — but you're looking for the "elbow" where improvement sharply diminishes.
🎮 Elbow Method Explorer — Hover over the curve
Hover over a point to see what K=N means for your data.
📉 Inertia (WCSS)
Within-Cluster Sum of Squares — the sum of squared distances from each point to its cluster centroid. Lower = tighter clusters.

K=1 → one big cluster, very high inertia.
K=N → every point is its own cluster, inertia=0.
📊 Silhouette Score
An alternative metric ranging from -1 to +1. Measures how similar a point is to its own cluster vs neighbouring clusters.

Score near +1 = well separated.
Score near 0 = on cluster boundary.
Score near -1 = wrong cluster.
Python — Elbow Method + Silhouette Score
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score
from sklearn.preprocessing import StandardScaler
import matplotlib.pyplot as plt

# Scale features first — critical for K-Means
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

inertias = []
silhouettes = []
K_range = range(2, 11)

for k in K_range:
    km = KMeans(n_clusters=k, n_init=10, random_state=42)
    labels = km.fit_predict(X_scaled)
    inertias.append(km.inertia_)
    silhouettes.append(silhouette_score(X_scaled, labels))

# Plot elbow curve
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 4))
ax1.plot(K_range, inertias, 'bo-')
ax1.set_xlabel('K'); ax1.set_ylabel('Inertia'); ax1.set_title('Elbow Method')
ax2.plot(K_range, silhouettes, 'gs-')
ax2.set_xlabel('K'); ax2.set_ylabel('Silhouette'); ax2.set_title('Silhouette Score')
plt.tight_layout(); plt.show()
🎯
K-Means++ Initialisation
Smarter starting centroids
Standard K-Means picks centroids randomly — this can lead to slow convergence or poor local minima. K-Means++ (the sklearn default) picks initial centroids that are spread out, dramatically improving results.
K-Means++ Initialisation Logic
✅
In sklearn, KMeans(init='k-means++') is the default. You get K-Means++ automatically. It's almost always better than random initialisation — no reason to switch it off.
⚠️
Strengths & Limitations
When K-Means works and when it fails
✅ K-Means Works Well When…
• Clusters are roughly spherical and equally sized
• Features have been scaled
• You have a rough idea of K
• Dataset is large (it scales as O(n·K·I) — fast)
• Clusters don't heavily overlap
❌ K-Means Struggles When…
• Clusters have irregular shapes (crescents, rings)
• Clusters have very different sizes or densities
• There are outliers (they pull centroids)
• K is unknown and hard to estimate
• Data has high dimensionality (curse of dimensionality)
🌍 Real World Example
Tesco Clubcard — Customer Segmentation with K-Means
Tesco uses clustering on Clubcard data (purchase frequency, basket size, category spend, recency) to segment millions of customers. K-Means with K≈8–12 segments identifies groups like "young families buying baby products", "health-conscious shoppers", "bulk buyers". Each segment receives tailored promotions, which are tracked by uplift in spend. The clustering runs monthly on updated data — segments shift as customer behaviour evolves. Features are scaled and outliers (extreme bulk buyers) are handled separately before clustering.

🌲 Hierarchical Clustering
Hierarchical clustering builds a tree of clusters — a dendrogram — that lets you see relationships at every level of granularity. No need to choose K upfront; cut the tree wherever makes sense.
⏱ ~30 min
🎯 2 Quizzes
⭐ 140 XP
🎮 Interactive
🌳
Agglomerative vs Divisive
Two directions to build the tree
⬆️
Agglomerative (Bottom-Up)
Start with every point as its own cluster. Repeatedly merge the two closest clusters until one cluster remains.

This is the sklearn default (AgglomerativeClustering) and by far the most common approach.
⬇️
Divisive (Top-Down)
Start with all points in one cluster. Repeatedly split the largest cluster until every point is its own cluster.

Less common in practice — computationally expensive and rarely implemented in standard libraries.
🎮 Interactive Dendrogram — click a cut level to see clusters
The horizontal dashed line shows where you "cut" the tree. Higher cut = fewer clusters.
Click on the dendrogram to set the cut height and see how many clusters result.
🔗
Linkage Methods
How to measure distance between clusters
When merging clusters, we need to define "distance between two clusters" — not just two points. The linkage method determines this. Your choice dramatically affects the shape of clusters found.
🎮 Explore Linkage Methods — click to compare
LinkageDistance = ?Cluster ShapeSensitivity to Outliers
SingleMinimum distance between any two pointsElongated, chainingVery high ⚠️
CompleteMaximum distance between any two pointsCompact, sphericalHigh
AverageMean distance between all point pairsBalancedModerate
Ward ⭐Minimises total within-cluster varianceCompact, equal-sizedLow ✅
⭐
Ward linkage is almost always the best default. It minimises the increase in total within-cluster variance at each merge step — producing compact, well-separated clusters. Use linkage='ward' in sklearn.
💻
Implementation
Hierarchical clustering in sklearn + scipy
Python — Agglomerative Clustering + Dendrogram
from sklearn.cluster import AgglomerativeClustering
from sklearn.preprocessing import StandardScaler
from scipy.cluster.hierarchy import dendrogram, linkage
import matplotlib.pyplot as plt

# Scale features first
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Fit agglomerative clustering (choose K after viewing dendrogram)
model = AgglomerativeClustering(
    n_clusters=4,         # set after inspecting dendrogram
    linkage='ward'         # recommended default
)
labels = model.fit_predict(X_scaled)

# Plot dendrogram using scipy (to help choose n_clusters)
Z = linkage(X_scaled, method='ward')
plt.figure(figsize=(12, 5))
dendrogram(Z,
    truncate_mode='level',
    p=5,                 # show last 5 merges
    color_threshold=8.0  # cut height — adjust based on your data
)
plt.title('Dendrogram (Ward Linkage)')
plt.xlabel('Sample Index')
plt.ylabel('Distance')
plt.axhline(y=8.0, color='red', linestyle='--', label='Cut height')
plt.legend(); plt.show()
✅ Advantages
• No K required upfront — choose after viewing dendrogram
• Produces a full hierarchy — view at any granularity
• Deterministic — no random initialisation
• Works well with Ward linkage on many real datasets
❌ Disadvantages
• Slow on large datasets — O(n² log n) with Ward
• Can't "undo" a bad merge — greedy algorithm
• Memory intensive for very large n
• Struggles with clusters of very different densities
🌍 Real World Example
Genomics — Gene Expression Clustering
Hierarchical clustering is the standard tool in genomics for discovering gene co-expression patterns. Researchers cluster thousands of genes by expression profile across conditions (e.g. cancer vs healthy tissue). The dendrogram reveals which genes "behave similarly" — a cluster of co-expressed genes often indicates shared biological function or regulatory pathway. Ward linkage is used because it keeps clusters compact. The dendrogram is cut at a height chosen by the researcher to yield biologically meaningful groups, often confirmed against known gene ontology databases.

🔵 DBSCAN
Density-Based Spatial Clustering of Applications with Noise. DBSCAN finds clusters of arbitrary shape, handles outliers naturally, and requires no upfront K — it discovers the number of clusters from the data.
⏱ ~30 min
🎯 2 Quizzes
⭐ 150 XP
🎮 Interactive
📍
Core Concepts: ε and MinPts
The two parameters that define everything
DBSCAN only has two hyperparameters, but they control everything. Understanding them is the key to using DBSCAN effectively.
🔵 ε (epsilon)
Neighbourhood Radius
The maximum distance between two points for them to be considered neighbours. A point's "ε-neighbourhood" is the circle of radius ε around it.

Too small → everything is noise. Too large → everything merges into one cluster.
🔢 MinPts
Minimum Points
The minimum number of points required within ε-distance for a point to be a core point. This controls minimum cluster density.

Rule of thumb: MinPts ≥ 2 × features. For 2D data: MinPts = 4 is common.
🎮 DBSCAN Point Types — hover over each point
DBSCAN classifies every point as Core, Border, or Noise.
Hover over a point to learn its classification.
🔵 Core Point
Has ≥ MinPts neighbours within ε-distance (including itself). These are the "dense" regions that form the heart of clusters.
🟢 Border Point
Has < MinPts neighbours within ε but is within ε of a core point. On the edge of a cluster — included but not a cluster seed.
🔴 Noise Point
Not a core point and not within ε of any core point. Labelled -1 by sklearn. These are your outliers — DBSCAN finds them automatically!

How DBSCAN Works
The algorithm step by step
Python — DBSCAN
from sklearn.cluster import DBSCAN
from sklearn.preprocessing import StandardScaler
import numpy as np

# Scale features — DBSCAN is distance-based
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

db = DBSCAN(
    eps=0.5,          # ε — neighbourhood radius
    min_samples=5    # MinPts — min neighbours to be core point
)
labels = db.fit_predict(X_scaled)

# labels == -1 means noise/outlier
n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
n_noise    = list(labels).count(-1)
print(f'Clusters: {n_clusters}, Noise points: {n_noise}')

# Choosing ε: use k-distance graph (k = MinPts)
from sklearn.neighbors import NearestNeighbors
nbrs = NearestNeighbors(n_neighbors=5).fit(X_scaled)
distances, _ = nbrs.kneighbors(X_scaled)
distances = np.sort(distances[:, -1])  # 5th nearest neighbour distance
# Plot distances — look for the "elbow" to choose ε

🗺️ t-SNE
t-Distributed Stochastic Neighbour Embedding. t-SNE takes high-dimensional data — hundreds or thousands of features — and maps it into 2D or 3D for visualisation. It preserves local structure: nearby points in high-D stay nearby in 2D.
⏱ ~30 min
🎯 2 Quizzes
⭐ 140 XP
🎮 Interactive
🌐
What t-SNE Does
From high dimensions to 2D — preserving neighbourhoods
Imagine you have 500 gene expression measurements per patient — a 500-dimensional dataset. You can't visualise that. t-SNE compresses it into 2D while trying to keep the neighbourhood relationships intact: patients that were similar in 500D should still cluster together in 2D.
⚠️
t-SNE is for visualisation only — not for creating features for downstream models. The 2D coordinates are not meaningful on their own; only the relative positions (neighbourhood) matter. Do not use t-SNE output as model input.
⚙️
How t-SNE Works (Intuitively)
Probabilities, not distances
1️⃣
High-D Similarities
For every pair of points in high-D, compute a probability that they are neighbours (using a Gaussian centred at each point). Close pairs → high probability.
2️⃣
2D Similarities
Randomly initialise 2D positions. Compute 2D neighbourhood probabilities using a t-distribution (heavier tails than Gaussian — stops crowding).
3️⃣
Minimise KL Divergence
Use gradient descent to move 2D points so the 2D probabilities match the high-D probabilities. The result: local neighbourhoods are preserved.

Perplexity controls how many neighbours each point considers. Low = local, High = global structure.

Rules for Interpreting t-SNE
Common mistakes to avoid
🎮 t-SNE Myths — click each card to reveal the truth
❓ MYTH
Cluster sizes in t-SNE reflect the real sizes of groups in the data.
✅ TRUTH
FALSE. Cluster sizes and distances between clusters in t-SNE are not meaningful. Only whether points are in the same neighbourhood matters.
❓ MYTH
The distance between two clusters in t-SNE tells you how different they are.
✅ TRUTH
FALSE. Global distances in t-SNE are not preserved. Two clusters far apart in 2D might be similar in high-D. Only local structure is reliable.
❓ MYTH
You can use t-SNE output coordinates as features to train a classifier.
✅ TRUTH
FALSE. t-SNE output is stochastic and non-deterministic. New points cannot be embedded into existing t-SNE space. Use PCA for feature extraction instead.
❓ MYTH
t-SNE with perplexity=5 and perplexity=50 will show the same structure.
✅ TRUTH
FALSE. Perplexity dramatically changes the result. Always try multiple values (5, 30, 50) and look for consistent patterns across runs. Never trust a single t-SNE plot.
Python — t-SNE Best Practices
from sklearn.manifold import TSNE
from sklearn.preprocessing import StandardScaler
import matplotlib.pyplot as plt

# Step 1: Scale first
X_scaled = StandardScaler().fit_transform(X)

# Step 2 (optional but recommended): PCA first if >50 features
# Reduces noise and speeds up t-SNE significantly
from sklearn.decomposition import PCA
X_pca = PCA(n_components=50).fit_transform(X_scaled)

# Step 3: Run t-SNE — try multiple perplexity values
for perp in [5, 20, 50]:
    tsne = TSNE(
        n_components=2,
        perplexity=perp,
        n_iter=1000,
        random_state=42  # set for reproducibility
    )
    X_2d = tsne.fit_transform(X_pca)

    plt.figure(figsize=(8, 6))
    plt.scatter(X_2d[:,0], X_2d[:,1], c=labels, cmap='tab10', alpha=0.6, s=15)
    plt.title(f't-SNE (perplexity={perp})')
    plt.colorbar(); plt.show()
🗺️ t-SNE
• Preserves local structure (neighbourhoods)
• Non-linear — captures complex manifolds
• Slow: O(n² log n) — use on ≤ 50k points
• Stochastic — different each run
• Global distances not meaningful
• Only for visualisation
📊 PCA (for comparison)
• Preserves global variance
• Linear — may miss non-linear structure
• Fast: O(n·d²) — scales well
• Deterministic — same result every time
• Can be used as model input
• Good for feature extraction + preprocessing
🌍 Real World Example
Single-Cell RNA Sequencing — Discovering Cell Types
t-SNE revolutionised single-cell genomics. Researchers measure the expression of ~20,000 genes in each of thousands of individual cells — impossibly high-dimensional. t-SNE compresses this to 2D, revealing islands of cells with similar gene expression. Each island corresponds to a cell type (T-cells, B-cells, stem cells). Before t-SNE, identifying these populations required expert manual gating. Now, the clusters appear visually, and researchers label them by the known marker genes of each island. This technique has been applied to map the entire human immune system.
