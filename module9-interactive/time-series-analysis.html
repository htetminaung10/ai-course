Intro to Time Series 📈
Every time you check stock prices, weather forecasts, or web traffic — you're working with time series. Learn what makes them unique and how Python handles time.
⏰ Datetime📅 DatetimeIndex🔁 Resampling📊 Real-world data

What is a Time Series?
Data with a time dimension — and why that changes everything
Definition
A time series is a sequence of data points collected or recorded at successive points in time, typically at uniform intervals. Unlike cross-sectional data (a snapshot), time series data has an inherent temporal ordering that carries information.

The order matters. Shuffling a time series destroys its meaning — unlike shuffling rows of a tabular dataset, which is harmless.
🌤️ Meteorology
Daily temperature, rainfall, wind speed. Forecasting weather 7 days ahead. Climate trends over decades.
💹 Finance
Stock prices (OHLCV), exchange rates, interest rates. Algorithmic trading, risk models, volatility forecasting.
🏭 Operations
Demand forecasting, inventory management, energy consumption, web traffic, server load monitoring.
Show series:
🧩
Key Characteristics
What makes time series special compared to regular tabular data
📉 Autocorrelation
Past values predict future values. Today's temperature is correlated with yesterday's. This temporal dependence violates the i.i.d. (independent, identically distributed) assumption that most ML models make.
🌊 Non-stationarity
Many time series have changing statistical properties over time — rising trends, seasonal swings, sudden shocks. Most classical models require stationarity first.
📅 Seasonality
Patterns that repeat at fixed periods: daily (peak traffic at 9am), weekly (low sales on Sunday), annual (Christmas retail spike).
📏 Missing Data
Gaps in time series are tricky — you can't just drop or fill randomly. Methods: forward-fill, interpolation, or explicitly modelling gaps. The position in time matters for imputation.

resampling.py
# Resampling: change the frequency of a time series
# Downsampling: high → low frequency (e.g., daily → monthly)
monthly = df['close'].resample('ME').mean()   # month-end mean
monthly_max = df['close'].resample('ME').max()    # month-end max

# Upsampling: low → high frequency (e.g., monthly → daily)
daily = monthly.resample('D').interpolate(method='linear')

# Common frequency aliases
# 'D' = calendar day, 'B' = business day
# 'h' = hour, 'min' = minute, 's' = second
# 'W' = weekly, 'ME' = month end, 'QE' = quarter end
# 'YE' = year end

# Rolling windows — for smoothing and features
df['ma_7']  = df['close'].rolling(7).mean()   # 7-day moving average
df['std_7'] = df['close'].rolling(7).std()    # 7-day rolling std

# Shifting — create lag features
df['lag_1'] = df['close'].shift(1)   # yesterday's close
df['lag_7'] = df['close'].shift(7)   # close from 7 days ago

resample() 👆 tap
Resampling groups a time series by a new frequency and applies an aggregation (mean, sum, max, etc.). Like groupby() for time. Downsampling reduces points; upsampling requires interpolation to fill new points.
rolling() 👆 tap
Rolling windows compute a statistic over a sliding window of N observations. A 7-day rolling mean is the mean of the last 7 days. Essential for smoothing noisy data and creating lag-based features for ML models.
shift() 👆 tap
Shifting moves the time series forward or backward by N periods, creating lag features. shift(1) gives yesterday's value. Critical for preventing data leakage — you must only use past data to predict future values.
diff() 👆 tap
Differencing computes the change between consecutive observations: y'ₜ = yₜ − yₜ₋₁. Key technique for removing trends and achieving stationarity. diff(12) computes seasonal differences (e.g., same month last year).

Time Series Decomposition 🔬
Every time series is a mixture of signals. Decomposition separates them — making patterns visible, anomalies obvious, and forecasting tractable.
📉 Trend🔄 Seasonality🌀 Residuals📊 ACF/PACF
🧩
The Three Components
Every time series = Trend + Seasonality + Residual
Core Idea
Observed = Trend + Seasonal + Residual (additive)  |  Observed = Trend × Seasonal × Residual (multiplicative)
Decomposition isolates each signal so you can analyse, model, and forecast each separately. It also helps diagnose whether a series is stationary and detect anomalies hiding in the noise.
Show layer:
📉 Trend
The long-run direction of the series — upward, downward, or flat. Can be linear or non-linear. Extracted by smoothing methods (moving average, lowess). Examples: population growth, long-term sales decline, climate warming.
🔄 Seasonality
Regular, repeating patterns at fixed periods: daily (commute peaks), weekly (lower sales Sunday), annual (Christmas spike). The period T is known in advance. Distinct from cycles, which have variable periods.
🌀 Residual / Noise
What remains after removing trend and seasonality. Should be random white noise if the decomposition was successful. Systematic patterns in residuals indicate missing structure (e.g. cycles, holidays, regime changes).
➕✖️
Additive vs Multiplicative
Which model fits your data?
➕ Additive Model
y(t) = Trend(t) + Season(t) + Residual(t)

Use when seasonal amplitude stays roughly constant over time — the summer peak is about the same size every year regardless of the overall level.

Example: temperature anomalies, birth rates.
✖️ Multiplicative Model
y(t) = Trend(t) × Season(t) × Residual(t)

Use when seasonal amplitude grows proportionally with the trend — the Christmas peak gets bigger as the company grows.

Example: retail sales, airline passengers. Can log-transform to convert to additive.
decomposition.py
from statsmodels.tsa.seasonal import seasonal_decompose
import matplotlib.pyplot as plt

# Additive decomposition (constant seasonal amplitude)
result = seasonal_decompose(df['sales'], model='additive', period=12)

# Access each component
trend    = result.trend      # smoothed trend
seasonal = result.seasonal   # seasonal component
residual = result.resid      # residuals

# Plot all four panels
result.plot()
plt.tight_layout()
plt.show()

# STL decomposition — more robust, handles outliers better
from statsmodels.tsa.stl._stl import STL
stl = STL(df['sales'], period=12, robust=True)
res = stl.fit()
📌STL (Seasonal-Trend decomposition using Loess) is generally preferred over classical decomposition — it handles any seasonal period, is robust to outliers, and allows the seasonal component to change over time. Statsmodels and Prophet both use it internally.

Autocorrelation: ACF & PACF
Measuring how much the past predicts the future
What is Autocorrelation?
The correlation of a series with a lagged version of itself. ACF at lag k measures how similar y(t) is to y(t−k). High ACF at lag 12 means the series repeats with a 12-period cycle (annual seasonality in monthly data).

ACF and PACF plots are diagnostic tools used to identify the order (p, q) of ARIMA models — one of the most important skills in classical time series analysis.
📊 ACF (Autocorrelation Function)
Correlation between y(t) and y(t−k) for each lag k — including indirect correlations through intermediate lags.

MA(q) pattern: ACF cuts off sharply after lag q
AR(p) pattern: ACF decays gradually (tails off)
Seasonal pattern: spikes at multiples of the seasonal period
📉 PACF (Partial ACF)
Correlation between y(t) and y(t−k) with the influence of intermediate lags removed.

AR(p) pattern: PACF cuts off sharply after lag p
MA(q) pattern: PACF decays gradually (tails off)

Rule of thumb: ACF tells you MA order; PACF tells you AR order.
acf_pacf.py
from statsmodels.graphics.tsaplots import plot_acf, plot_pacf
import matplotlib.pyplot as plt

fig, axes = plt.subplots(2, 1, figsize=(12, 6))

plot_acf(df['sales'], lags=40, ax=axes[0])
plot_pacf(df['sales'], lags=40, ax=axes[1], method='ywm')
plt.tight_layout()
plt.show()

# Blue shaded region = 95% confidence interval
# Bars extending beyond = statistically significant autocorrelation

Understanding ARIMA(p,d,q)
Three parameters, each with a specific job
p — AR Order
Number of autoregressive lags to include. Uses past p values of y to predict the next value.

Read from: PACF plot (number of significant lags before cutoff).

AR(2): y(t) = c + φ₁y(t-1) + φ₂y(t-2) + ε(t)
d — Differencing Order
Number of times to difference the series to achieve stationarity. d=0: already stationary. d=1: first difference removes linear trend. d=2: removes quadratic trend.

Read from: ADF test and visual inspection.
q — MA Order
Number of moving average terms (past forecast errors) to include.

Read from: ACF plot (number of significant lags before cutoff).

MA(1): y(t) = c + ε(t) + θ₁ε(t-1)

d (I): 1
q (MA): 0
arima_fit.py
from statsmodels.tsa.arima.model import ARIMA
from statsmodels.tsa.statespace.sarimax import SARIMAX

# Fit ARIMA(1,1,1)
model = ARIMA(df['sales'], order=(1,1,1))
result = model.fit()
print(result.summary())

# Forecast 12 steps ahead
forecast = result.forecast(steps=12)
conf_int = result.get_forecast(12).conf_int()  # 95% CI

# SARIMA(1,1,1)(1,1,1,12) — seasonal ARIMA
# (p,d,q) × (P,D,Q,s) where s = seasonal period
sarima = SARIMAX(df['sales'],
                  order=(1,1,1),
                  seasonal_order=(1,1,1,12))
sarima_result = sarima.fit(disp=False)

# Auto-select best ARIMA order using AIC
import pmdarima as pm
auto = pm.auto_arima(df['sales'], seasonal=True, m=12,
                       stepwise=True, suppress_warnings=True)
print(auto.summary())

Exponential Smoothing
Simple, powerful, and widely used in production
Core Idea
Exponential smoothing weights recent observations more heavily than older ones — the influence of past data decays exponentially.
Unlike ARIMA (which models autocorrelation structure), Exponential Smoothing directly models the level, trend, and seasonality of a series. The Holt-Winters method is still a top performer in forecasting competitions.
SES — Simple
Single Exponential Smoothing — only for level (no trend, no seasonality).

ŷ(t+1) = αy(t) + (1−α)ŷ(t)

α (0–1) controls how fast memory decays. α=0.9: highly reactive. α=0.1: slow/smooth.
Holt's — Level + Trend
Double Exponential Smoothing — handles trended series.

Two smoothing equations: one for level (α), one for trend (β). Projects the trend forward linearly.

Also called Holt's Linear Method.
Holt-Winters — Full
Triple Exponential Smoothing — level + trend + seasonality.

Three parameters: α (level), β (trend), γ (seasonal). Additive or multiplicative seasonal variant. Widely used in production systems.

Stationarity & Statistical Testing 🧪
The prerequisite for ARIMA — understand what stationarity means, how to test for it formally, and how to achieve it through differencing and transformations.
📏 Stationarity🧪 ADF Test➖ Differencing📐 Log Transform
📏
What is Stationarity?
The property almost all classical models require
Formal Definition
A time series is strictly stationary if its joint probability distribution is the same for any shift in time. In practice we use weak (covariance) stationarity, which requires three conditions:

1. Constant mean: E[y(t)] = μ for all t (no trend)
2. Constant variance: Var[y(t)] = σ² for all t (no volatility changes)
3. Constant autocovariance: Cov[y(t), y(t+k)] depends only on lag k, not on t
❌ Non-Stationary Signs
• Visible upward or downward trend
• Increasing or decreasing variance over time
• Unit root (random walk behaviour)
• ACF decays very slowly to zero
• Mean and variance change across subperiods
✅ Stationary Signs
• Fluctuates around a constant mean
• Consistent variance throughout
• ACF drops quickly to near zero
• Statistical properties look similar in any window
• Passes the ADF or KPSS test
🧪
The ADF Test
Augmented Dickey-Fuller — the standard unit root test
Hypothesis
H₀: Series has a unit root (non-stationary)  |  H₁: Series is stationary
If the p-value < 0.05, reject H₀ — the series is stationary. If p-value > 0.05, fail to reject — the series likely has a unit root and needs differencing or transformation.
🧪 ADF Test Simulator — click a series to run the test
stationarity.py
from statsmodels.tsa.stattools import adfuller
import numpy as np

def adf_test(series, name=''):
    result = adfuller(series, autolag='AIC')
    print(f'ADF Test: {name}')
    print(f'  ADF Statistic : {result[0]:.4f}')
    print(f'  p-value       : {result[1]:.4f}')
    print(f'  Stationary    : {result[1] < 0.05}')

# Step 1: test original series
adf_test(df['sales'], 'Original')

# Step 2: if non-stationary, first difference
adf_test(df['sales'].diff().dropna(), 'First Difference')

# Step 3: log transform first (if variance grows with level)
adf_test(np.log(df['sales']).diff().dropna(), 'Log + Diff')

Prophet & Forecasting Evaluation 🔮
Facebook's Prophet makes production-grade forecasting accessible — and knowing how to evaluate your forecasts separates good practitioners from great ones.
🔮 Prophet📊 MAE/RMSE/MAPE🔭 Cross-validation📉 Baseline
🔮
Facebook Prophet
Designed for business forecasting at scale
What is Prophet?
Prophet (Meta, 2017) is an open-source forecasting library designed for business time series — daily data with strong seasonal patterns, holidays, and missing values. Under the hood it fits a decomposable model:

y(t) = trend(t) + seasonality(t) + holidays(t) + ε(t)

Prophet uses Stan (a probabilistic programming language) internally and provides automatic changepoint detection, multiple seasonalities, and uncertainty intervals out of the box.
📈 Trend Models
Two options: linear (with optional saturation cap) or logistic growth (for metrics with natural limits). Automatically detects changepoints — moments where the trend rate changes. You can supply known changepoints or let Prophet find them.
📅 Seasonality
Uses Fourier series to model seasonality at multiple periods simultaneously: yearly (365.25d), weekly (7d), daily (24h). You can add custom seasonalities for domain-specific cycles (e.g. academic year, sports season).
🎉 Holidays
Pass a DataFrame of holiday dates — Prophet learns the effect of each holiday as a separate additive component. Built-in country holiday calendars via the holidays package. Essential for retail forecasting.
prophet_forecast.py
from prophet import Prophet
import pandas as pd

# Prophet requires columns named 'ds' (date) and 'y' (value)
df_prophet = df.reset_index().rename(columns={'date':'ds', 'sales':'y'})

# Basic model — works out of the box
model = Prophet(
    yearly_seasonality=True,
    weekly_seasonality=True,
    daily_seasonality=False,
    changepoint_prior_scale=0.05   # flexibility of trend changepoints
)
model.fit(df_prophet)

# Create future dataframe and forecast
future = model.make_future_dataframe(periods=365)
forecast = model.predict(future)

# Plot decomposed components
model.plot_components(forecast)

# Add UK public holidays
from prophet.make_holidays import make_holidays_df
holidays = make_holidays_df(year_list=[2022,2023,2024], country='UK')
model_h = Prophet(holidays=holidays)
model_h.fit(df_prophet)

Time Series Cross-Validation
You cannot use random k-fold — time must flow forward
Walk-Forward Validation
Standard k-fold cross-validation randomly shuffles data — catastrophic for time series because future data leaks into training. Instead, use walk-forward (expanding window) or rolling window validation:

Expanding window: Train on [1…t], test on [t+1…t+h]. Extend training window by 1 step each iteration. Simulates real production forecasting.

Rolling window: Train on a fixed-size window [t-w…t], test on [t+1…t+h]. Useful when the model should not rely on very old data (concept drift).
prophet_cv.py
from prophet.diagnostics import cross_validation, performance_metrics

# Walk-forward CV built into Prophet
df_cv = cross_validation(
    model,
    initial='730 days',   # minimum training size
    period='180 days',    # how often to cut off
    horizon='365 days'    # forecast horizon to evaluate
)

# Get MAE, RMSE, MAPE, etc. at each horizon
metrics = performance_metrics(df_cv)
print(metrics[['horizon','mae','rmse','mape']].head())
