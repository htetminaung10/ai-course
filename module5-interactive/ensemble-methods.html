🎪 Introduction to Ensemble Methods
Why ask one expert when you can ask a thousand? Ensemble methods combine multiple models to produce predictions stronger than any individual model — the mathematical formalisation of "wisdom of crowds."

The Wisdom of Crowds
The intuition behind why ensembles work
🐄 The Cow-Weighing Experiment
In 1906, Francis Galton visited a county fair where 800 people guessed the weight of an ox. No single expert got it right — but the median of all 800 guesses was 1,207 lbs. The ox weighed 1,198 lbs.

The crowd was more accurate than any individual, including the livestock experts. This is the core insight behind ensemble ML: diverse, partially-correct models combined outperform any single model.

Each "weak learner" alone is only slightly better than random. But together...
🗂️
Three Ensemble Strategies
👜 Bagging
Bootstrap Aggregating
Train models in parallel on random subsets of data. Average predictions.

Reduces variance. Good for high-variance models (trees).
Example: Random Forest
⬆️ Boosting
Train models sequentially. Each new model focuses on the mistakes of the previous one.

Reduces bias. Converts weak learners into strong ones.
Example: AdaBoost, XGBoost
🏗️ Stacking
Train diverse base models, then train a meta-learner on their predictions.

Most flexible. Can combine any model types.
Example: Blended model pipelines
🌍 Real World Example
Kaggle Competition Winners — Almost Always Use Ensembles
Since 2012, the top solutions in nearly every Kaggle ML competition use ensemble methods. The 2015 Netflix Prize ($1M) was won with a blend of 107 models. Modern winning solutions typically stack XGBoost, LightGBM, Neural Networks, and Linear models — each capturing different patterns — with a meta-learner combining them. Individual models plateau around 85% accuracy; well-constructed ensembles routinely push to 92%+.

👜 Bagging — Bootstrap Aggregating
Train dozens of models in parallel on random subsamples of your data, then average their predictions. This simple trick dramatically reduces variance without increasing bias.
⏱ ~30 min
🎯 2 Quizzes
⭐ 160 XP
🎮 Interactive
🔁
Bootstrap Sampling
Sampling with replacement — the key ingredient
🔁 What is "Sampling with Replacement"?
Given a dataset of N samples, create a new dataset of N samples by:
1. Pick a random sample → add to new dataset
2. Put it back → it can be picked again
3. Repeat N times

Result: each bootstrap sample has the same size as the original, but ~63% unique samples (the rest are duplicates). About 37% of original samples are out-of-bag (OOB) — never selected.
🎲 Interactive Bootstrap — click to resample
Original dataset (10 samples):
✅ Why Bagging Works
Each model sees a slightly different dataset → learns slightly different patterns → makes different errors. When you average predictions, random errors cancel out but systematic signal remains.

Mathematically: averaging n uncorrelated models with variance σ² gives variance σ²/n.
📦 Out-of-Bag (OOB) Evaluation
The ~37% samples not used for training each model can be used to evaluate that model for free. Average OOB scores across all models = an unbiased performance estimate — like free cross-validation!

Set oob_score=True in sklearn.
🌍 Real World Example
Bagging for Medical Image Diagnosis
Oxford's medical AI group trained 50 CNN models on different bootstrap samples of chest X-rays to detect pneumonia. A single CNN had 87% accuracy and was brittle — small image rotations changed predictions. The bagged ensemble averaged 94% accuracy and was robust to rotations: each CNN had learned different orientations from its bootstrap sample. The OOB scores gave reliable performance estimates without needing a separate validation set.
Python — Bagging
from sklearn.ensemble import BaggingClassifier
from sklearn.tree import DecisionTreeClassifier

# Bag 50 decision trees, each trained on a random 80% subsample
bag = BaggingClassifier(
    estimator=DecisionTreeClassifier(max_depth=None),  # deep trees = high variance
    n_estimators=50,
    max_samples=0.8,       # 80% of training samples per tree
    max_features=1.0,       # all features (RF uses a subset)
    bootstrap=True,         # sampling WITH replacement
    oob_score=True,         # free evaluation on out-of-bag samples
    random_state=42
)
bag.fit(X_train, y_train)

print(f"Test accuracy: {bag.score(X_test, y_test):.3f}")
print(f"OOB accuracy:  {bag.oob_score_:.3f}")  # free performance estimate

🌲 Random Forests
Random Forest = Bagging + Feature Randomness. By restricting each tree to a random subset of features at every split, trees become decorrelated — and the ensemble becomes dramatically stronger.
⏱ ~35 min
🎯 2 Quizzes
⭐ 180 XP
🎮 Interactive
🎲
Bagging vs Random Forest
One extra trick makes all the difference
👜 Plain Bagging
Each tree uses all features for every split. Problem: if one feature is very strong, all trees will use it at the root → trees are correlated → averaging correlated models doesn't reduce variance as much.
🌲 Random Forest
At each split, only a random subset of features (typically √p for classification, p/3 for regression) is considered. This forces trees to use different features → trees are decorrelated → averaging works much better.
⚙️ Random Forest Hyperparameter Explorer

📊
Feature Importance
Random Forests reveal which features matter most
📊 Mean Decrease in Impurity (MDI)
Every time a feature is used in a split, track how much it reduces Gini impurity, averaged across all trees. Features used more often in higher-level splits get higher importance.

This gives you a free feature ranking from every Random Forest — useful for feature selection and model understanding.
🌍 Real World Example
NHS Patient Readmission Prediction
The NHS uses Random Forests to predict 30-day hospital readmission risk. The model uses 40+ features — diagnoses, medications, ward stays, lab results. Feature importance reveals the top predictors: prior admission count (28%), discharge destination (18%), number of medications (14%), age (11%). This guides both model improvement and clinical insight: "patients discharged to care homes with 5+ prior admissions need a follow-up call within 3 days."
Python — Random Forest
from sklearn.ensemble import RandomForestClassifier
import pandas as pd

rf = RandomForestClassifier(
    n_estimators=200,       # number of trees (more = better, diminishing returns)
    max_depth=None,         # grow full trees (bagging handles variance)
    max_features='sqrt',   # √p features per split (classification default)
    min_samples_leaf=2,    # minimum samples per leaf
    oob_score=True,         # out-of-bag score
    n_jobs=-1,              # use all CPU cores
    random_state=42
)
rf.fit(X_train, y_train)
print(f"Test: {rf.score(X_test, y_test):.3f}, OOB: {rf.oob_score_:.3f}")

# Feature importances
importances = pd.Series(rf.feature_importances_, index=X_train.columns)
print(importances.sort_values(ascending=False).head(10))

⬆️ Boosting
While bagging builds trees in parallel and averages, boosting builds them sequentially — each tree focuses on fixing the mistakes of all previous trees. This turns weak learners into strong ones.
⏱ ~35 min
🎯 2 Quizzes
⭐ 180 XP
🎮 Interactive
🎯
AdaBoost — Adaptive Boosting
Invented by Freund & Schapire, 1995 — won the Gödel Prize
🎯 The AdaBoost Algorithm
Step 1: Train a weak classifier on the data (all samples equally weighted)
Step 2: Identify misclassified samples → increase their weight
Step 3: Train the next classifier, paying more attention to hard cases
Step 4: Combine all classifiers, weighted by their individual accuracy

Repeat for T iterations. Final prediction = weighted vote of all T classifiers.
🎮 Watch weights evolve — click NEXT to advance rounds
🌍 Real World Example
Face Detection — Viola-Jones Algorithm
Every camera from 2001 onwards used a boosted ensemble for face detection (Viola-Jones). Each "weak learner" was a single Haar-like feature comparison — dead simple. But by AdaBoosting 200 of them, the detector achieved real-time face detection on 2001 hardware. The boosting insight: the early rounds eliminate easy backgrounds, later rounds focus on the ambiguous edge cases. This was the first practical real-time face detector.
⚡ Bagging vs Boosting
Bagging: Parallel, independent trees. Reduces variance. Works even with strong learners. Harder to overfit.

Boosting: Sequential, dependent trees. Reduces bias. Uses weak learners. Can overfit if too many rounds.
⚠️ Boosting Pitfalls
• Sensitive to noisy data (outliers get huge weights)
• Can overfit with too many estimators
• Harder to parallelise (sequential nature)
• Slower to train than Random Forest
Python — AdaBoost
from sklearn.ensemble import AdaBoostClassifier
from sklearn.tree import DecisionTreeClassifier

# AdaBoost with decision stumps (depth=1) as weak learners
ada = AdaBoostClassifier(
    estimator=DecisionTreeClassifier(max_depth=1),  # decision stumps
    n_estimators=200,    # rounds of boosting
    learning_rate=0.1,   # shrinks contribution of each tree
    random_state=42
)
ada.fit(X_train, y_train)
print(f"Accuracy: {ada.score(X_test, y_test):.3f}")

# Monitor staged performance (accuracy vs n_estimators)
import numpy as np
staged_scores = [ada.score(X_test, y_test)

🚀 Gradient Boosting & XGBoost
XGBoost won more Kaggle competitions than any other algorithm. It's gradient boosting with regularisation, parallel processing, and built-in handling of missing values — industrial-strength ensemble learning.
⏱ ~40 min
🎯 2 Quizzes
⭐ 200 XP
📐
Gradient Boosting — the core idea
📐 From AdaBoost to Gradient Boosting
AdaBoost reweights samples. Gradient Boosting generalises this: instead of reweighting, each new tree is trained to predict the residual errors (pseudo-residuals) of all previous trees.

It's literally applying gradient descent in function space: each tree is a "step" that nudges predictions in the direction that most reduces the loss.

This works for any differentiable loss function — not just classification.
Gradient Boosting — Residual Fitting
F₀(x) = initial prediction (mean of y)
r₁ = y − F₀(x)    (residuals after round 0)
h₁(x) = tree fit to r₁
F₁(x) = F₀(x) + η·h₁(x)  (η = learning rate)
r₂ = y − F₁(x)    (residuals after round 1)
h₂(x) = tree fit to r₂
⋮
F_T(x) = F₀(x) + η·h₁(x) + η·h₂(x) + ... + η·h_T(x)
🚀
Why XGBoost Dominates
🛡️ Regularisation
L1 (alpha) and L2 (lambda) regularisation built in — penalises complex trees. Controls overfitting without needing max_depth alone.
⚡ Speed
Column subsampling, parallel tree building, cache-aware computation. 10-100x faster than sklearn's GradientBoosting on large datasets.
🔧 Missing Values
Automatically learns the best direction for missing values at each split. No manual imputation needed for XGBoost.
🌍 Real World Example
Higgs Boson Discovery at CERN
The 2014 Higgs Boson Kaggle challenge was solved with XGBoost. Physicists at CERN needed to identify Higgs boson decay signals from 11 million particle collision events with 30 features. XGBoost achieved 3.806 AMS (competition metric) vs 3.782 for Deep Neural Networks, in a fraction of the training time. The XGBoost paper (Chen & Guestrin, 2016) has been cited 60,000+ times and the algorithm now runs in AWS, Google Cloud, and Azure ML as a built-in primitive.

🏗️ Stacking
Train diverse base models, then train a meta-learner on their predictions. Stacking lets models specialise — the meta-learner learns when to trust which model, often outperforming any single approach.
⏱ ~30 min
🎯 2 Quizzes
⭐ 160 XP
🏗️
How Stacking Works
🏗️ Two-Layer Stacking
Layer 1 — Base models: Train diverse models (RF, XGBoost, Logistic Reg, SVM) on the training data. Each makes predictions on the validation set.

Layer 2 — Meta-learner: Train a simple model (Logistic Regression is common) on the base models' predictions as features. It learns which base model to trust for which type of input.

Key trick: use out-of-fold predictions for layer 1 to prevent leakage.
🌍 Real World Example
Netflix Prize — 107-Model Blended Ensemble
The 2009 Netflix Prize ($1M) was won by "BellKor's Pragmatic Chaos" — a 107-model stacked ensemble. Base models included SVD matrix factorisation, RBMs, Nearest Neighbour methods, and time-series models. The meta-learner (linear regression on OOF predictions) learned that SVD was best for users with many ratings, while neighbourhood methods excelled for new users. The ensemble achieved 10.06% RMSE improvement over Netflix's baseline — the threshold was 10%. Individual models maxed out at ~8%.
Python — Stacking
from sklearn.ensemble import StackingClassifier
from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier
from sklearn.linear_model import LogisticRegression
from sklearn.svm import SVC

# Diverse base models (diversity = key to stacking performance)
base_models = [
    ('rf',  RandomForestClassifier(n_estimators=200, random_state=42)),
    ('xgb', GradientBoostingClassifier(n_estimators=200, random_state=42)),
    ('svc', SVC(probability=True, kernel='rbf')),
]

# Simple meta-learner (Logistic Regression is often best)
meta = LogisticRegression(C=0.1)

stack = StackingClassifier(
    estimators=base_models,
    final_estimator=meta,
    cv=5,              # out-of-fold predictions (prevents leakage)
    passthrough=False,  # don't include original features in meta-learner
)
stack.fit(X_train, y_train)
print(stack.score(X_test, y_test))
