∂ Calculus Foundations

Derivatives, the chain rule, and partial differentiation — the mathematical engine behind neural network training.

Why Neural Networks Need Calculus

Training a neural network is an optimisation problem: find the weights that minimise a loss function. Calculus tells us which direction to move each weight to reduce the loss.

The Core Idea

A derivative measures rate of change. If loss L depends on weight w, then dL/dw tells us: for a tiny increase in w, how much does L change?

  • Positive derivative → increasing w increases loss → go left
  • Negative derivative → increasing w decreases loss → go right
  • Zero derivative → local minimum or maximum

What You Need

  • Power rule — differentiate polynomials
  • Chain rule — differentiate compositions f(g(x))
  • Partial derivatives — differentiate w.r.t. one variable at a time
  • Gradient — vector of all partial derivatives

The Chain Rule — Key for Backpropagation

Neural networks are compositions of functions. The chain rule lets us differentiate them layer by layer.

If y = f(g(x)), then dy/dx = f'(g(x)) · g'(x)

Neural net example: If z = w·x, a = σ(z), L = (a−y)², then dL/dw = dL/da · da/dz · dz/dw — each piece is simple, and we multiply them together (backpropagation!).

Chain Rule Step-by-Step

f(x) = (3x² + 1)⁵ — click each step

① Identify
② Outer f'
③ Inner g'
④ Multiply
Identify the composition:
Let u = g(x) = 3x² + 1   (inner function)
Then f(u) = u⁵   (outer function)
So f(g(x)) = (3x² + 1)⁵
Differentiate the outer function (treat inner as a variable):
f'(u) = 5u⁴  →  f'(g(x)) = 5(3x² + 1)⁴
Differentiate the inner function:
g(x) = 3x² + 1  →  g'(x) = 6x
Multiply the pieces:
dy/dx = f'(g(x)) · g'(x) = 5(3x² + 1)⁴ · 6x = 30x(3x² + 1)⁴

Partial Derivatives & the Gradient

With multiple inputs (like all the weights in a network), we differentiate w.r.t. each variable separately — treating all others as constants.

∂f/∂x means: differentiate f w.r.t. x, holding all other variables constant

Example

f(x, y) = x²y + 3y²

∂f/∂x = 2xy   (y is constant)
∂f/∂y = x² + 6y   (x is constant)

The Gradient

The gradient ∇f is the vector of all partial derivatives:

∇f = [∂f/∂x, ∂f/∂y] = [2xy, x²+6y]

It points in the direction of steepest ascent. Going opposite to it gives the steepest descent — exactly what gradient descent does.

0.5
∇f at (1.0, 0.5) = [1.00, 4.00]

DeepMind AlphaFold 2 — calculus solving the 50-year protein folding problem
Proteins are chains of amino acids that fold into 3D shapes determining their biological function. Predicting those shapes had stumped scientists for 50 years. DeepMind's AlphaFold 2 solved it in 2020 using a deep neural network trained with backpropagation — the same chain rule of calculus you just learned. 🕸️ Neural Network Architecture

Layers, weights, forward pass, and activation functions — how information flows through a network.

What is a Neural Network?

A neural network is a function approximator made of layers of neurons. Each neuron computes a weighted sum of its inputs, adds a bias, then passes the result through an activation function.

output = activation( w₁x₁ + w₂x₂ + … + wₙxₙ + b )
📥
Input Layer

Receives raw features (pixels, numbers, embeddings). No computation — just passes values forward.

🧠
Hidden Layers

Where learning happens. Each layer learns increasingly abstract representations of the data.

📤
Output Layer

Produces the prediction. Softmax for classification, linear for regression, sigmoid for binary.

The Forward Pass — Step by Step

In a forward pass, data flows left-to-right through every layer. Click each step:

① Input
② Linear
③ Activate
④ Stack
⑤ Output
Input x — e.g. a 28×28 MNIST image flattened to a vector of 784 pixel values, each normalised to [0,1].
Linear transformation: z = Wx + b
W is the weight matrix (shape: neurons_out × neurons_in), b is the bias vector. This is a matrix multiplication.
Activation: a = f(z)
Apply element-wise: ReLU(z) = max(0, z), or sigmoid(z) = 1/(1+e⁻ᶻ). Adds non-linearity — without this, stacking layers is equivalent to one linear layer.
Stack layers: the output a of one layer becomes the input x to the next.
z² = W²a¹ + b², a² = f(z²), z³ = W³a² + b³, …
Output layer: the final layer uses a task-specific activation:
• Softmax → multi-class probabilities (sum to 1)
• Sigmoid → binary probability [0,1]
• Linear → regression (unbounded)

Activation Functions

ReLU(x) = max(0, x)

Default choice for hidden layers. Fast, sparse, avoids vanishing gradient. Can "die" (output 0 forever) for negative inputs.

Weights, Biases & Parameters

Parameter Count

For a fully-connected layer with nᵢₙ inputs and nₒᵤₜ outputs:

params = nᵢₙ × nₒᵤₜ + nₒᵤₜ (biases)

A tiny network [784 → 128 → 64 → 10] has:

  • 784×128 + 128 = 100,480
  • 128×64 + 64 = 8,256
  • 64×10 + 10 = 650
  • Total: 109,386 params

Weight Initialisation

Bad initialisation causes vanishing/exploding gradients. Common strategies:

  • Xavier/Glorot — for sigmoid/tanh, scale by √(2/(nᵢₙ+nₒᵤₜ))
  • He init — for ReLU, scale by √(2/nᵢₙ)
  • Zeros — never! All neurons learn same thing (symmetry problem)
  • Random normal — small variance (e.g. σ=0.01)

⛰️ Optimisation & Training

Gradient descent, backpropagation, loss functions, and the full training loop.

Loss Functions — Measuring Error

The loss (cost) function measures how wrong the model is. Training minimises it. Choose based on your task:

L = (1/n) Σ(yᵢ − ŷᵢ)²

Mean Squared Error — regression tasks. Penalises large errors heavily (squared). Smooth gradient everywhere.

Gradient Descent — Descending the Loss Landscape

We move weights in the opposite direction of the gradient. The learning rate controls step size.

Backpropagation — Computing Gradients Efficiently

Backprop applies the chain rule layer-by-layer from output to input, reusing intermediate computations. Click each phase:

① Forward
② Compute Loss
③ Output δ
④ Hidden δ
⑤ Update W
Forward pass: compute all activations a¹, a², …, aᴸ storing them for later.
z¹ = W¹x + b¹ → a¹ = σ(z¹) → z² = W²a¹ + b² → a² = softmax(z²)
Compute loss: e.g. Cross-Entropy L = −Σ yᵢ log(ŷᵢ)
This is a scalar — the single number we want to minimise.
Output layer error signal δᴸ:
δᴸ = ∂L/∂zᴸ = ŷ − y (for softmax + cross-entropy, this simplifies nicely)
This tells us: in which direction should the last layer's pre-activations change?
Propagate backward to hidden layers:
δˡ = (Wˡ⁺¹)ᵀ δˡ⁺¹ ⊙ f'(zˡ)
Transpose of weight matrix routes error back; element-wise multiply by activation derivative.
Update weights and biases:
∂L/∂Wˡ = δˡ (aˡ⁻¹)ᵀ
Wˡ ← Wˡ − η · ∂L/∂Wˡ
bˡ ← bˡ − η · δˡ

Optimiser Variants

Click each to explore — vanilla gradient descent is rarely used in practice.

📉
SGD
Stochastic Gradient Descent: update on each mini-batch. Noisy but fast. w ← w − η·∇L
⚡
Momentum
Adds velocity: v ← βv + ∇L, w ← w − ηv. Accelerates in consistent gradient directions, dampens oscillations.
📊
RMSProp
Adapts learning rate per-parameter using running average of squared gradients. Good for RNNs.
👑
Adam
Combines Momentum + RMSProp. Maintains both first and second moment estimates. Default choice for most tasks.
📅
LR Schedule
Decay η over training: step decay, cosine annealing, warm restarts. Helps convergence to sharp minima.
📦
Batch Size
32–256 typical. Larger = more stable gradient estimate but needs more memory. Smaller = noisier but can generalise better.

The Training Loop

① Load Batch
② Forward
③ Loss
④ Backward
⑤ Update
⑥ Validate
Sample a mini-batch of (x, y) pairs from the training set. Shuffle each epoch to prevent ordering effects.
ŷ = model(x) — run the forward pass, storing activations at every layer.
L = loss(ŷ, y) — compute scalar loss value. Also track running average for logging.
loss.backward() — PyTorch/TF automatically computes ∂L/∂w for every weight via autograd.
optimizer.step() then optimizer.zero_grad() — apply the weight update, then clear gradients for the next batch.
After each epoch, evaluate on the held-out validation set. Track val_loss and val_accuracy. Stop early if val_loss stops improving (early stopping).

Regularisation — Preventing Overfitting

L2 / Weight Decay

L_total = L + λ Σ wᵢ²

Penalises large weights. Encourages simpler models. Built into most optimisers as weight_decay.

Dropout

During training: zero each neuron with prob p

Randomly drops neurons during training (typically p=0.5). Forces the network to learn redundant representations. Disabled at inference.

Batch Normalisation

x̂ = (x − μ_B) / √(σ²_B + ε)

Normalises activations within each mini-batch. Stabilises training, allows higher learning rates, acts as mild regulariser.

Early Stopping

Stop when val_loss stops improving

Monitor validation loss. If it hasn't improved for N epochs (patience), restore the best checkpoint and stop training.

🌍 Real World Example
OpenAI training GPT-4 — optimisation at a scale that cost $100 million
Training GPT-4 reportedly cost over $100 million and used thousands of NVIDIA A100 GPUs running for months. The optimiser was Adam (Adaptive Moment Estimation) — a variant of gradient descent that adapts the learning rate individually for each of GPT-4's ~1.8 trillion parameters. A naive SGD with a fixed learning rate would either diverge (rate too high) or take centuries to converge (rate too low). Adam's key insight: parameters that receive large gradients frequently should have their learning rate reduced; sparse parameters should receive a larger update. Without adaptive optimisation, training a model at this scale would be computationally impossible. The choice of optimiser — SGD vs Adam vs AdaGrad — is not academic. At GPT-4 scale, the wrong choice costs tens of millions of dollars.

🔍 Convolutional Neural Networks

Convolutions, filters, pooling, and landmark architectures for image understanding.

Why Not Just Use Fully-Connected Layers for Images?

The Parameter Problem

A 224×224 RGB image has 150,528 inputs. A single FC layer with 1,000 neurons = 150 million parameters — just for one layer. Unscalable and prone to overfitting.

Spatial Structure is Thrown Away

Flattening an image loses all spatial relationships. A pixel at (10,10) is no longer "near" the pixel at (11,10). FC layers cannot exploit this structure.

CNNs exploit two key priors about images: local connectivity (nearby pixels are related) and translational invariance (a cat is a cat wherever it appears). This gives parameter sharing — the same filter is applied everywhere.

Key CNN Concepts — Click to Reveal

👣
Stride
How many pixels the filter moves each step. Stride 2 halves the output spatial size. Larger stride = smaller feature map.
📐
Padding
"Same" padding adds zeros around the border so output size = input size. "Valid" = no padding (output shrinks).
📚
Filter Depth
Each filter has depth = number of input channels (e.g. 3 for RGB). Output channels = number of filters applied.
⬇️
Max Pooling
Downsamples by taking the max value in each 2×2 window. Reduces spatial size, provides translation invariance.
👁️
Receptive Field
The region of the input a neuron "sees". Grows with depth: early layers see local textures, deep layers see whole objects.
➡️
Flatten → FC
After conv+pool layers, the feature map is flattened to a vector then fed into fully-connected layers for final classification.

🔁 RNNs, LSTMs & Graph Neural Networks

Sequence models, vanishing gradients, LSTM gates, and message passing on graphs.

Why Sequence Models?

Standard MLPs and CNNs assume inputs are independent and identically distributed. But many real-world problems have sequential structure — the order matters.

📝
Text

"The bank by the river" vs "The bank that holds money" — word meaning depends on context.

📈
Time Series

Stock prices, sensor readings, weather — each value depends on previous values.

🎵
Audio / Speech

Speech recognition, music generation — temporal patterns span many time steps.

Recurrent Neural Networks — Unrolled View

An RNN processes sequences one step at a time, passing a hidden state hₜ forward. The same weights W are reused at every step.

hₜ = tanh(Wₕhₜ₋₁ + Wₓxₜ + b)
5

Shared weights: Wₕ and Wₓ are the same at every time step. This makes RNNs parameter-efficient for sequences of any length — but also causes the vanishing gradient problem.

The Vanishing Gradient Problem

When backpropagating through time (BPTT), gradients are multiplied by Wₕ at each step. If |Wₕ| < 1, gradients shrink exponentially — early time steps receive nearly zero gradient.

0.8

⚠️ When |Wₕ| > 1 gradients explode (gradient clipping fixes this). When |Wₕ| < 1 they vanish (LSTMs fix this with the cell state highway).

LSTM Gates — Click Each Gate

Long Short-Term Memory networks add a cell state Cₜ that flows with minimal modification — a "gradient highway". Three learned gates control information flow.

fₜ = σ(Wf·[hₜ₋₁, xₜ] + bf)

Forget gate: decides what to erase from cell state. Output is between 0 (forget all) and 1 (keep all). σ = sigmoid.

GRU — A Simpler Alternative

LSTM

  • 3 gates: forget, input, output
  • Separate cell state Cₜ and hidden state hₜ
  • ~4× more parameters than vanilla RNN
  • Better for very long sequences

GRU (Gated Recurrent Unit)

  • 2 gates: reset and update
  • No separate cell state
  • Fewer parameters → faster training
  • Comparable performance on most tasks
zₜ = σ(Wz·[hₜ₋₁,xₜ]) (update gate)
rₜ = σ(Wr·[hₜ₋₁,xₜ]) (reset gate)

Graph Neural Networks

Many real-world problems are graphs: social networks, molecules, knowledge graphs, road networks. GNNs extend deep learning to graph-structured data.

Message Passing

Each node aggregates messages from its neighbours, then updates its own representation:

mᵥ = AGG({hᵤ : u ∈ N(v)})
hᵥ' = UPDATE(hᵥ, mᵥ)

AGG can be sum, mean, or max. After K layers, each node sees K-hop neighbourhood.

GNN Applications

  • Node classification — predict properties of individual nodes (e.g. paper topic in citation network)
  • Link prediction — will two nodes connect? (recommender systems)
  • Graph classification — predict molecule toxicity
  • Traffic forecasting — road network as graph

Allows decoder to "look back" at all encoder states, not just the last one. Score(q,k) = qᵀk/√d. Foundation of Transformers.

🌍 Real World Example
Google Translate and Spotify — sequence models powering 500 million users daily
Google Translate (2016–2023) used a sequence-to-sequence LSTM architecture to translate 100+ languages. The encoder LSTM read the entire input sentence, compressing it into a hidden state vector (a fixed-length "thought vector" representing meaning). The decoder LSTM then unrolled this vector into the target language word by word. The vanishing gradient problem you just learned about was why early RNNs failed on long sentences — the gradient for "She" at position 1 was essentially zero by the time backpropagation reached it from position 30. LSTMs' gates (forget, input, output) fixed this by maintaining a cell state that carries information across arbitrarily long sequences. Meanwhile, Spotify's music recommendation uses a Graph Neural Network: songs are nodes, user co-listens are edges. GNN aggregates neighbourhood features — songs frequently played together share embedding space — enabling Spotify to generate the "Discover Weekly" playlist for 500 million users every Monday morning.
