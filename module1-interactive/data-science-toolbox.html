1.1 Variables & Data Types
Every piece of data in Python has a type
🧠 What is a Variable?
Think of a variable as a labelled box in your computer's memory. You put a value inside, give it a name, and you can use that name later to get the value back — or change it.

Python uses dynamic typing: you don't need to declare the type. Python figures it out automatically.
🌍 Real World Example
Netflix's Recommendation Engine
Every time Netflix stores a user's age (age = 28), their username (name = "Alice"), or their watch history count (watched = 347), they're using variables. Data scientists at Netflix then use these variables to build models that predict what you'll watch next.

1.2 Conditionals & Loops
Make decisions and repeat actions
🔀 if / elif / else — Making Decisions
Conditionals let your code choose a path based on whether something is true or false. Think of them as the "if this happens, do that" logic of real life.

Python uses indentation (4 spaces) to define code blocks — no curly braces like other languages!
🌍 Real World Example
Fraud Detection at Banks
Barclays uses conditions like: "If the transaction amount > £5000 AND the location is abroad AND it's past midnight, flag as suspicious." This is exactly if/elif/else logic running millions of times per second.
Python — Conditionals
transaction_amount = 7500
is_abroad = True
hour = 2   # 2 AM

if transaction_amount > 5000 and is_abroad and hour < 6:
    print("🚨 Flagged as suspicious!")
elif transaction_amount > 5000:
    print("⚠️  Large transaction — review required")
else:
    print("✅ Transaction approved")

# Output: 🚨 Flagged as suspicious!
🔁 for Loops — Repeat Actions
A for loop lets you iterate over a collection and perform an action on each item. Instead of writing the same code 1000 times, write it once in a loop.

A while loop repeats as long as a condition is True.
Python — Loops
# for loop — iterate over a list
exam_scores = [72, 88, 65, 94, 77]
total = 0

for score in exam_scores:
    total += score

print(f"Average: {total / len(exam_scores):.1f}%")  # 79.2%

# List comprehension — pythonic one-liner
passed = [s for s in exam_scores if s >= 70]
print(f"Students who passed: {len(passed)}")  # 4

# range() — generate a sequence of numbers
for i in range(1, 6):
    print(f"Week {i}: training model...")

1.3 Python Functions
Package reusable logic into named blocks
⚙️ Why Functions?
Functions follow the DRY principle: Don't Repeat Yourself. Instead of writing the same 10 lines repeatedly, define them once as a function and call it whenever needed.

In data science, functions help you standardise preprocessing steps, build reusable pipelines, and make code testable.
🌍 Real World Example
Spotify's Audio Feature Extraction
Spotify's engineers write a function extract_audio_features(song) that computes tempo, energy, danceability, etc. This same function is called for every song in their 100M+ library — write once, use millions of times.
Python — Functions
# Basic function: def name(parameters): → return result
def calculate_bmi(weight_kg, height_m):
    """Calculates Body Mass Index. Weight in kg, height in metres."""
    bmi = weight_kg / (height_m ** 2)
    return round(bmi, 1)

print(calculate_bmi(70, 1.75))  # 22.9

# Default parameters
def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

print(greet("Alice"))             # Hello, Alice!
print(greet("Bob", "Good morning")) # Good morning, Bob!

# Lambda — small anonymous functions (used heavily in Pandas!)
square = lambda x: x ** 2
print(square(7))  # 49

Why Environments?
The "dependency hell" problem — and how to solve it
🔥 The Problem Without Conda
Imagine you're working on two projects simultaneously:

• Project A (old system): requires numpy==1.20
• Project B (new system): requires numpy==1.26

Without environments, installing one breaks the other. Conda solves this by creating isolated sandboxes — each project lives in its own environment with its own packages.
Without Conda: System Python └─ numpy 1.26 ← Project B works ✅ ← Project A BROKEN ❌ With Conda: Environment: project_a Environment: project_b └─ Python 3.8 └─ Python 3.11 └─ numpy 1.20 ✅ └─ numpy 1.26 ✅ └─ pandas 1.3 ✅ └─ pandas 2.0 ✅ └─ scikit-learn 0.24 ✅ └─ scikit-learn 1.3 ✅
🌍 Real World Example
AI Teams at Google DeepMind
Research teams at DeepMind use environment management so that the TensorFlow 1.x models from 2018 still run correctly alongside their new JAX-based systems. Without this, years of research code would be unmaintainable.
⌨️
Essential Conda Commands
The commands you'll use every day
1
Create a new environment

Terminal — Conda Commands
# Create a new environment
conda create --name ds_env python=3.11

# Activate it (Mac/Linux)
conda activate ds_env

# Install core data science packages
conda install numpy pandas scikit-learn matplotlib jupyter

# List all your environments
conda env list

# Export to share with teammates
conda env export > environment.yml

# Recreate from a shared file
conda env create -f environment.yml

# Deactivate (exit environment)
conda deactivate

🌿 Git & Version Control
Track changes, collaborate without chaos, and never lose your work again. Git is the time machine every developer — and data scientist — needs.
⏱ ~30 min
🎯 3 Quizzes
⭐ 150 XP
🛠 Essential
🕰️
What is Git?
A distributed version control system
🕰️ Git = Time Machine for Code
Git records a history of every change you make. You can go back to any point in time, create parallel "alternate timelines" (branches) to try new ideas, and merge them back safely.

Without Git, teams use nightmare approaches like report_v1.py, report_FINAL.py, report_FINAL_v2_ACTUAL.py... 😱
🌍 Real World Example
NHS AI Diagnostic Models
A team at the NHS building AI for radiology uses Git so that: (1) if a model update reduces accuracy, they can instantly roll back; (2) multiple radiologists and engineers can work simultaneously without overwriting each other's work; (3) every model change is auditable — critical for medical regulatory compliance.
🗂️
The Git Workflow
Your Files Staging Area Local Repo Remote (GitHub) │ │ │ │ │ git add file.py │ │ │ │ ──────────────────►│ │ │ │ │ git commit -m "" │ │ │ │ ──────────────────►│ │ │ │ │ git push │ │ │ │ ──────────────────►│ │ │ │ git pull │ │ │ │ ◄──────────────────│
Terminal — Essential Git Commands
## SETUP (do once)
git config --global user.name  "Your Name"
git config --global user.email "you@email.com"

## START A PROJECT
git init                    # initialise a new repo in current folder
git clone <url>             # copy an existing repo from GitHub

## DAILY WORKFLOW
git status                  # see what's changed
git add analysis.ipynb      # stage a specific file
git add .                   # stage ALL changes
git commit -m "Add EDA for sales data"  # save a snapshot
git push origin main        # send to GitHub
git pull                    # get latest from teammates

## BRANCHES — parallel timelines
git branch feature/new-model    # create a branch
git checkout feature/new-model  # switch to it
git checkout -b hotfix          # create AND switch (shortcut)
git merge feature/new-model     # bring changes back to main

## HISTORY & UNDO
git log --oneline           # see commit history
git diff                    # see unstaged changes
git checkout -- file.py     # discard changes to a file
⚠️
Golden Rules: (1) Commit often — small commits are easier to understand. (2) Write meaningful commit messages: "Fix data leakage in preprocessing" not "fix stuff". (3) Never force-push to shared branches.
🌿 Branching Strategy for Data Science
A common pattern for data science teams:

• main — production-ready, always works
• develop — integration branch for new work
• feature/experiment-name — try a new model or approach

This way, you can experiment freely on a feature branch without breaking the working model.

Why NumPy?
100x faster than Python lists for maths
⚡ The Speed Problem with Python Lists
Python lists are flexible but slow for maths. To multiply every element by 2, you need a loop. NumPy arrays use vectorisation — the operation applies to all elements simultaneously, using optimised C code under the hood.

For 1 million numbers: Python list loop ≈ 500ms, NumPy ≈ 2ms — 250x faster!
🌍 Real World Example
Image Processing at Instagram
A 1080p photo is stored as a NumPy array of shape (1080, 1920, 3) — 1080 rows, 1920 columns, 3 colour channels (RGB). Instagram's filters apply NumPy operations across millions of pixels in milliseconds. Every deep learning framework (PyTorch, TensorFlow) is built on NumPy arrays.

Python — NumPy Essentials
import numpy as np

## Creating arrays
a = np.array([10, 20, 30, 40, 50])   # from list
z = np.zeros((3, 4))                # 3×4 array of zeros
r = np.random.rand(100)             # 100 random values 0–1
s = np.arange(0, 100, 10)           # [0,10,20,...,90]

## Array properties
print(a.shape)    # (5,)  ← shape
print(a.dtype)    # int64 ← data type
print(a.ndim)     # 1     ← dimensions

## Vectorised operations (apply to ALL elements at once)
print(a * 2)      # [20, 40, 60, 80, 100]
print(a + 5)      # [15, 25, 35, 45, 55]
print(a ** 2)     # [100, 400, 900, 1600, 2500]

## Slicing — just like Python lists but for N dimensions
print(a[1:4])     # [20, 30, 40] (index 1 to 3)
print(a[-1])      # 50 (last element)

## Boolean masking — powerful filtering!
prices = np.array([10.5, 25.0, 8.2, 42.1, 15.0])
expensive = prices[prices > 15]   # [25.0, 42.1]

## Statistics — built-in and fast
print(np.mean(prices))   # 20.16
print(np.std(prices))    # 12.49
print(np.max(prices))    # 42.1
print(np.argmax(prices)) # 3 (index of max)

DataFrames & Series
The two core data structures
📋 Series
A 1D labelled array. Like a single column in a spreadsheet. Has an index (labels) and values.
📊 DataFrame
A 2D labelled table. Like a spreadsheet or SQL table. Multiple Series sharing the same index.
🌍 Real World Example
UK COVID-19 Vaccination Data
The NHS published COVID vaccination data as a CSV. A data scientist loads it with pd.read_csv('vaccinations.csv') and instantly has a DataFrame with columns like date, doses_given, region, age_group. From there, groupby operations reveal which regions lagged behind, filtering finds under-vaccinated demographics, and merge joins it with population data to calculate coverage percentages.

df = pd.read_csv("customers.csv")

Data Cleaning
Real data is messy — fix it before analysis
⚠️
Data Quality Fact: Data scientists spend ~80% of their time on data cleaning. Pandas makes this manageable — but you still need to know what you're looking for: missing values, duplicates, wrong types, outliers.
Python — Data Cleaning
## Missing values
df.isnull().sum()              # count nulls per column
df.dropna()                    # drop ALL rows with any null
df["age"].fillna(df["age"].mean())  # fill nulls with mean

## Duplicates
df.duplicated().sum()          # count duplicate rows
df.drop_duplicates()           # remove them

## Data types
df["salary"] = df["salary"].astype(float)
df["date"] = pd.to_datetime(df["date"])  # parse dates

## String cleaning
df["city"] = df["city"].str.strip().str.title()  # "  london " → "London"

## Rename columns
df.rename(columns={"salary": "annual_salary"}, inplace=True)
📊
GroupBy — Split, Apply, Combine
The most powerful Pandas operation
🔬 GroupBy: The Excel PivotTable but better
GroupBy splits your data into groups, applies a function to each group, then combines the results. The pattern is: split → apply → combine.

Example: "What's the average salary per city?" — group by city, then take the mean of salary.
Python — GroupBy & Merge
## GroupBy — split-apply-combine
avg_salary_by_city = df.groupby("city")["salary"].mean()
# city
# Bristol       38000
# London        62666
# Manchester    48000

## Multiple aggregations at once
city_stats = df.groupby("city").agg({
    "salary": ["mean", "max", "count"],
    "age":    "mean"
})

## Merge (like SQL JOIN)
orders = pd.read_csv("orders.csv")
merged = pd.merge(df, orders, on="customer_id", how="left")
#  how = "inner", "left", "right", "outer"

## Apply custom function to each row/column
def salary_band(salary):
    if salary < 40000: return "Junior"
    elif salary < 60000: return "Mid"
    else: return "Senior"

df["band"] = df["salary"].apply(salary_band)
